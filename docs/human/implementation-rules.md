# Implementation Rules

Graphene реализуется как дизайн-система для Vue 3.

Основные правила:

- использовать Vue 3;
- использовать TypeScript;
- писать компоненты как `.vue` Single File Components;
- в `.vue` компонентах использовать `<script setup lang="ts">`;
- типизировать `props`, `emits` и публичные типы компонентов;
- предпочитать Composition API и `<script setup>`;
- держать локальные стили компонента в `<style scoped>`;
- выносить общие значения в токены;
- делить токены на базовые и семантические;
- в компонентах использовать семантические токены, а не базовые цвета напрямую;
- состояния компонентов описывать через семантические токены, например `--g-action-*` и `--g-field-*`;
- не задавать произвольные цвета, отступы, радиусы и размеры;
- считать `--g-radius-none` базовым скруглением для большинства элементов;
- использовать скругления только там, где это явно нужно по дизайну;
- не использовать inline styles без сильной причины;
- не смешивать компоненты дизайн-системы с бизнес-логикой приложений;
- не добавлять зависимости от внешнего интернета.

Компонент должен иметь понятную ответственность.
Если компонент становится слишком сложным, нужно выделять вложенные компоненты.

Новые компоненты должны попадать в правильную категорию внутри `src/components/`.
