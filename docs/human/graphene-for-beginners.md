# Graphene Для Новичка

Graphene - это набор правил, стилей и Vue-компонентов, чтобы разные приложения организации выглядели единообразно.

Если коротко: Graphene помогает не придумывать интерфейс заново в каждом проекте.

## Как Устроен Проект

`src/`

Главная папка Graphene. Здесь лежит то, что потом будут использовать приложения: токены, стили, шрифты и Vue-компоненты.

`src/tokens/`

Токены дизайна: цвета, размеры, отступы, радиусы, тени, z-index, типографика.

Токены делятся на два уровня:

- базовые токены - сырые значения, например конкретный цвет, размер, радиус или отступ;
- семантические токены - смысловые роли интерфейса, например основной текст, фон страницы, обычная граница, опасное действие.

Компоненты должны использовать семантические токены, а не придумывать случайные `13px` или новый цвет.

Скругления тоже задаются токенами: `none`, `xs`, `sm`, `md`, `lg`, `xl`, `2xl`, `full`.
Базовый характер Graphene - строгий, ближе к IBM Carbon: у большинства элементов скругления нет.

В Pixso у токенов используются режимы `light` и `dark`.
В коде это становится одной CSS-переменной с разными значениями для `[data-theme='light']` и `[data-theme='dark']`.

`src/styles/`

Глобальные стили Graphene.

Здесь подключается шрифт, reset, тема и базовый внешний вид обычных HTML-элементов. Даже если приложение пока не использует Vue-компоненты Graphene, подключение этих стилей уже должно приблизить интерфейс к общему виду Graphene.

`src/styles/base.css` - основная точка подключения базового внешнего вида Graphene.

`src/styles/fonts.css`

Подключение IBM Plex Sans из локальных файлов. Шрифт не должен загружаться из интернета.

`src/assets/`

Статические ресурсы самой Graphene.

Сюда попадает только то, что является частью дизайн-системы и нужно ее компонентам или стилям. Например, локальные шрифты.

`src/assets/fonts/`

Файлы IBM Plex Sans.

Это обязательный шрифт Graphene. Приложения должны работать без внешнего интернета, поэтому шрифт хранится локально.

`src/components/`

Vue-компоненты Graphene.

Это готовые элементы интерфейса, которые команды должны использовать вместо самописных кнопок, полей, таблиц, модалок и других повторяемых частей UI.

`src/components/base/`

Самые простые компоненты: кнопки, поля ввода, чекбоксы, иконки, бейджи.

Сейчас в проекте уже есть первые базовые компоненты:

- `BaseButton` - кнопка с вариантами `primary` и `secondary`, размерами `sm`, `md`, `lg` и состоянием `disabled`;
- `BaseCheckbox` - чекбокс без встроенного label, состояниями `unchecked`, `checked`, `disabled`;
- `BaseRadio` - радиокнопка без встроенного label, состояниями `unchecked`, `checked`, `disabled`;
- `BaseInput` - поле ввода с label, helper text, размерами `sm`, `md`, `lg` и состояниями `default`, `hover`, `focused`, `filled`, `invalid`, `disabled`;
- `BaseTextarea` - многострочное поле ввода с label, helper text и состояниями `default`, `hover`, `focused`, `filled`, `invalid`, `disabled`.

`BaseInput` поддерживает правую зону через slot `trailing`.
Так можно показать иконку поиска из `src/assets/icons/search.svg` без копирования иконки в приложение.
Для `type="search"` нативный browser clear-крестик отключен, чтобы справа оставались только элементы Graphene.

`src/components/layout/`

Компоненты для структуры экрана: панели, контейнеры, области, шапки, сайдбары.

`src/components/data/`

Компоненты для отображения данных: таблицы, списки, деревья, свойства, пагинация.

`src/components/feedback/`

Компоненты обратной связи: модальные окна, уведомления, ошибки, загрузка, пустые состояния.

Например, диалог удаления устройства должен быть использованием компонента из этой группы, а не отдельным правилом в проекте.

`src/components/navigation/`

Компоненты навигации: вкладки, меню, хлебные крошки, панели действий.

`docs/`

Документация Graphene.

Здесь лежат короткие правила и пояснения: как устроена система, как использовать шрифт, почему нельзя зависеть от интернета, как должна работать витрина.

`src/assets/icons/`

Отобранный локальный набор иконок Graphene.

`src/assets/icons/others/`

Большой фиксированный запасной набор иконок. Если иконка становится регулярной частью дизайн-системы, ее переносят в `src/assets/icons/`.

`examples/`

Витрина Graphene.

Здесь показываются примеры использования компонентов и типовых экранов. Это не отдельная копия дизайн-системы. Витрина должна импортировать токены, стили и компоненты из `src/`.

`examples/index.html`

Главная offline-страница витрины.

Ее можно открыть локально или разместить во внутреннем контуре организации.

`examples/src/views/`

Страницы витрины: примеры кнопок, форм, таблиц, диалогов и других частей интерфейса.

`examples/src/example-data/`

Тестовые данные только для витрины. Они не являются частью Graphene и не должны использоваться в реальных приложениях.

`docs/ai/`

Инструкции для AI-агентов, которые используют Graphene или помогают разрабатывать саму дизайн-систему.

## Что Мы Пока Не Выделяем

В проекте пока нет отдельных папок `patterns` и `composables`.

Их можно добавить позже, если появится реальная необходимость. Сейчас структура специально остается простой, чтобы не создавать лишние уровни раньше времени.

## Главная Идея

Порядок от простого к сложному такой:

1. Токены.
2. Глобальные стили.
3. Базовые компоненты.
4. Более сложные компоненты.
5. Примеры в витрине.

Чем ниже уровень, тем он фундаментальнее. Чем выше уровень, тем ближе он к реальному экрану приложения.
