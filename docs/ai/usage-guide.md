# AI Usage Guide

Этот документ читает AI, который создает интерфейс приложения с использованием Graphene.

Цель: не рисовать интерфейс с нуля, а собирать его из правил, стилей и компонентов Graphene.

## Перед Началом

1. Подключи глобальные стили Graphene через `src/styles/base.css`.
2. Используй шрифт IBM Plex Sans через Graphene, не подключай внешний шрифт.
3. Проверь, какие компоненты уже есть в `src/components/`.
4. Не создавай локальные аналоги компонентов Graphene без необходимости.

## Как Верстать

Используй уровни Graphene по порядку:

1. `src/tokens/` - базовые и семантические токены.
2. `src/styles/` - базовый внешний вид приложения.
3. `src/components/` - готовые Vue-компоненты.
4. `examples/` - примеры правильного применения.

`examples/index.html` - локальная статическая витрина, которая открывается напрямую в браузере без dev-сервера.
`cd examples && npm run dev` нужен только для разработки витрины.

Если есть компонент Graphene, используй его.
Если компонента нет, используй обычную HTML-разметку со стилями Graphene и семантическими токенами.
Если пример в `examples` показывает существующий компонент Graphene, импортируй компонент из `src/components/`, а не копируй его разметку и стили.

Текущие базовые компоненты:

- `BaseButton` - использовать для основных и вторичных действий;
- `BaseCheckbox` - использовать для одиночного boolean-переключателя без встроенного label;
- `BaseInput` - использовать для простого текстового ввода; правую зону действия передавать через slot `trailing`.

По умолчанию не добавляй скругления.
Базовая форма Graphene - прямоугольная, ближе к IBM Carbon.
Используй `--g-radius-none`, если нет явного правила или макета со скруглением.

## Запреты

Нельзя:

- подключать CDN;
- подключать внешние шрифты;
- использовать случайные цвета, если есть семантический токен;
- задавать произвольные отступы, если есть spacing token;
- писать inline styles без сильной причины;
- копировать код из `examples` как отдельную дизайн-систему;
- делать интерфейс только под широкий десктоп.

## Темы

Интерфейс должен работать в светлой и темной теме.

Переключение темы ожидается через:

`data-theme="light"`

или:

`data-theme="dark"`

Используй семантические токены, например `--g-text-primary`, а не базовые цвета вроде `--g-gray-950`.
Не меняй семантические токены Graphene и их привязку к базовым токенам в коде ради локального совпадения конкретного компонента или экрана.
Если токен не соответствует Pixso, скажи владельцу, какой semantic token нужно поправить в Pixso.
После правки в Pixso синхронизируй токены из Pixso в код, не выбирая base tokens вручную.

## Устройства

Интерфейс должен работать на десктопах, планшетах и телефонах.

Учитывай:

- портретную ориентацию телефона;
- touch-размеры интерактивных элементов;
- отсутствие лишнего горизонтального скролла;
- корректную работу меню, таблиц и модальных окон на узком экране;
- полноэкранную оболочку приложения.

## Offline/Intranet

Приложение должно работать без внешнего интернета.

Все шрифты, ассеты и runtime-зависимости должны быть локальными или доступными во внутреннем контуре организации.
